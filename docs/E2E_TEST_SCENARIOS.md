# End-to-End (E2E) Test Scenarios

## Overview

This document outlines 50 complete end-to-end workflows to be tested for the Yeko project. These cover the full user journeys from UI interactions through API calls to database operations.

## School Administration & Setup (1-10)

1. **Admin Registration**: Admin user registration and account verification.
2. **School Profile**: School profile creation with basic information (name, address, contact details).
3. **Academic Year**: Academic year configuration and term setup.
4. **Grade Structure**: Grade level/class structure definition.
5. **Classroom Management**: Classroom creation and capacity management.
6. **Subject Catalog**: Subject catalog creation and organization.
7. **Programs**: Academic program setup with coefficients.
8. **Fee Config**: School fee structure configuration.
9. **Payment Plans**: Payment plan template creation.
10. **Localization**: Initial system settings and localization setup.

## User Management & Roles (11-20)

1. **Teacher Creation**: Teacher account creation by admin.
2. **Teacher Profile**: Teacher profile completion and subject assignment.
3. **Student Enrollment**: Student enrollment with parent account creation.
4. **Student Profile**: Student profile setup with emergency contacts.
5. **Parent Linking**: Parent account linking to multiple students.
6. **RBAC Assignment**: Role-based permission assignment.
7. **Auth**: User authentication and login verification.
8. **Password Reset**: Password reset workflow.
9. **Deactivation**: Account deactivation and reactivation.
10. **Session Mgmt**: User session management and timeout handling.

## Academic Management (21-30)

1. **Class Setup**: Class creation and teacher assignment.
2. **Class Enrollment**: Student enrollment in specific classes.
3. **Timetables**: Timetable creation and scheduling.
4. **Attendance**: Daily attendance recording by teacher.
5. **Grading**: Grade entry and validation.
6. **Report Cards**: Report card generation and distribution.
7. **Progress Tracking**: Student progress tracking.
8. **Coefficients**: Subject coefficient application.
9. **Year Rollover**: Academic year rollover process.
10. **Curriculum**: Curriculum progress monitoring.

## Financial Management (31-40)

1. **Fee Assignment**: Fee structure assignment to students.
2. **Plan Selection**: Payment plan selection by parents.
3. **Installments**: Installment payment processing.
4. **Receipts**: Payment confirmation and receipt generation.
5. **Balances**: Outstanding balance tracking.
6. **History**: Payment history viewing.
7. **Refunds**: Refund request and processing.
8. **Financial Reports**: Financial report generation.
9. **Fiscal Years**: Fiscal year management.
10. **Audit**: Transaction audit trail verification.

## Teacher Workflows (41-50)

1. **Dashboard**: Teacher dashboard access and overview.
2. **Class Lists**: Assigned class list viewing.
3. **Roster**: Student roster management.
4. **Lesson Plans**: Lesson plan creation and updates.
5. **Assignments**: Assignment creation and student submission handling.
6. **Messaging**: Parent communication through messaging system.
7. **Conduct**: Conduct record management.
8. **Leave Requests**: Leave request submission and approval.
9. **Analytics**: Performance analytics viewing.
10. **Term Reports**: End-of-term report compilation.
